- **Same-day follow-up to 0.0.23: pinned down the exact root cause of codegraph's 92-vs-50 shortfall on the shared-base-method case, and confirmed it is NOT a `cg-trace.sh` script bug.** Prompted directly by the user asking whether the wrapper script itself might be at fault. Full methodology in `tool-collaboration-benchmark.md`'s new "Round 6" section.

  - Every fix above was live-verified two ways: a simulated `cbm_call()`/`cg_call()` error object piped through each script's exact new filter (proving the failure mode, then proving the fix), and a full regression pass of all five patched scripts' normal success paths against the still-indexed `RuoYi-Vue-Plus`/`plus-ui` projects (unchanged output, confirming no behavioral regression).
